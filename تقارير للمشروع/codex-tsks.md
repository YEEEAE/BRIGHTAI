قائمة المهام التقنية (Codex 5.3)

1.✅✅✅✅✅✅✅✅✅✅ مهمة فحص الروابط الداخلية وإصلاحها (Performance/SEO): 
إجراء فحص شامل لجميع الملفات الصفحية (HTML/JS) واكتشاف الروابط والمسارات المكسورة، ثم إصلاحها وضمان توحيد نمط الروابط الداخلية.  
   المدخلات: جميع ملفات الصفحات (HTML/CSS/JS) وقوائم الروابط الحالية.  
   المخرجات: تقرير بالروابط المكسورة قبل الإصلاح وبعده، وصفحات مصححة بدون أخطاء، وتحسين التصفح الداخلي.  


3.✅✅✅✅✅✅✅✅✅✅ مهمة توحيد شريط التنقل ونظام التصميم (UX): 
تطوير مكوّن واجهة موحّد (Nav/Search) يضم نفس العناصر والترتيب في كل صفحات الخدمات، بالإضافة إلى تصميم نظام تنسيق مشترك (CSS/HTML) لضمان مظهر وتجربة مستخدم متسقة.  
   المدخلات: ملفات التصميم الحالية (HTML/CSS/JS) والكائنات الرسومية (logos, buttons).  
   المخرجات: مكوّن Nav موحّد يحمل صندوق بحث، مكتبة تصميم مشتركة (Design System) قابلة لإعادة الاستخدام، وتحسين تجربة المستخدم العام.  

4. مهمة إصلاح المسارات المكسورة وتكرار السكربتات (Quality/Performance): ✅✅✅✅✅✅✅✅✅✅ 
تحليل ملفات الصفحات واكتشاف المسارات النسبية الخاطئة أو المكررة للسكريبتات والأصول (JS/CSS) وإصلاحها. كما يشمل إزالة أي تحميل مكرر للموارد ضمن الصفحة.  
   المدخلات: ملفات الصفحات (HTML/CSS/JS) وبنية الدلائل (assets).  
   المخرجات: مسارات صحيحة وموحدة للموارد، وتقارير فنية توضح عدد الأخطاء قبل وبعد الإصلاح.  

5. مهمة دمج الشيفرات الخلفية (Integration):✅✅✅✅✅✅✅✅✅✅ 
 مراجعة ودمج كود الخادم في مجلدي `backend/` و`server/` لإنشاء قاعدة شفرة واحدة (Single Source of Truth). يشمل ذلك توحيد نقاط النهاية المشتركة (chat, search, Groq) ومكونات التكوين والميدل وير.  
   المدخلات: المستودع الحالي للكود في `backend/` و`server/`.  
   المخرجات: مستودع موحد لخادم التطبيق، وقائمة بالملفات المحذوفة أو المدموجة، مع وثائق الدمج.  

6. مهمة نشر الشات بوت وربطه بخادم Groq (Integration/UX)✅✅✅✅✅✅✅✅✅✅ : 
تشغيل تطبيق الدردشة (Chatbot) في بيئة الإنتاج وربط واجهته بخادم Groq الآمن. يشمل ذلك تجميع الشيفرة إلى وحدة واحدة، واعتماد endpoint داخلي (مثل `/api/groq/stream`)، وإضافة حالات واجهة للمستخدم (تحميل/خطأ/إعادة محاولة)، وضمان حفظ محفوظات الجلسة بما يحترم الخصوصية.  
   المدخلات: كود الشات بوت الحالي، مفاتيح Groq، إعدادات API.  
   المخرجات: شات بوت متكامل يعمل عبر الخادم، مع تقرير اختبارات وظيفية لجميع السيناريوهات (نجاح، خطأ، انتهاء المهلة، حد المعدل).  

7. مهمة منصة بحث موحدة وفهرسة حيّة (UX):
 بناء نظام بحث داخلي مشترك لجميع صفحات الخدمات، مع فهرس صفحات يتم تحديثه تلقائيًا عند نشر محتوى جديد. يتحقق ذلك باستخدام محرك بحث مثل Elasticsearch أو Algolia وتحديث سريع للفهارس (Live Index) بدلاً من البحث الثابت.  
   المدخلات: بنية الصفحات الحالية ومحرك بحث معيّن.  
   المخرجات: واجهة بحث موحدة تعمل في كل الصفحات وتعيد نتائج دقيقة دون أخطاء 404.  

8. مهمة تحسين الأداء العميق لملفات JS/CSS (Performance):
 استخدام أدوات قياس الأداء (مثل Google Lighthouse وWebPageTest) لتحليل سرعة تحميل الصفحة ومؤشرات Core Web Vitals. ثم تطبيق تحسينات مثل تجميع وضغط (minify) الملفات، تفعيل التخزين المؤقت (Caching)، وتقنيات التحميل الكسول (lazy-loading) للصور والفيديو.  
   المدخلات: ملفات المصدر (JS/CSS/Images)، تقارير أداء أولية (Lighthouse/WebPageTest).  
   المخرجات: ملفات موارد مضغوطة ومحسّنة، وتقارير قبل/بعد تظهر تحسن مؤشرات الأداء مثل LCP وCLS وTTFB.  

9. مهمة توسيع SEO الدلالي (SEO): ✅✅✅✅✅✅✅✅✅✅
تحديث عناصر SEO التقنية والمحتوى الدلالي لاستهداف السوق المحلي. يشمل ذلك إصلاح روابط og:image، وضبط الـ canonical و hreflang في جميع صفحات الخدمات، وإضافة بيانات schema.org مناسبة (مثل BreadcrumbList، LocalBusiness)، وتحسين نصوص الـ alt للصور والكلمات المفتاحية ضمن المحتوى. كما يشمل تدقيق ملف sitemap.xml وحذف الروابط التالفة أو المنخفضة الجودة.  
   المدخلات: محتوى الموقع، خرائط الموقع الحالية، قائمة كلمات مفتاحية محلية.  
   المخرجات: صفحات محسّنة SEO (عناوين ووصف بديل، data-schema)، خريطة موقع محدثة، وتقارير أداة تدقيق الـ SEO.  

10. مهمة إعداد خطة القياس والتحليلات (Measurement):✅✅✅✅✅✅✅✅✅✅
 تكوين Google Analytics 4 لتعقب أهداف تحويل محددة (مثل إرسال نماذج التواصل، بدء الشات) وربطها بلوحات معلومات تفاعلية عبر Google Data Studio (Looker Studio). يتضمن ذلك إنشاء أحداث مخصصة (Custom Events) في GA4، وتكوين التقارير لعرض المقاييس (جلسات، تحويلات، السلوك).  
    المدخلات: حسابات Google Analytics/Ads، تصميم متطلبات KPIs.  
    المخرجات: إعداد تقارير GA4 مسجلة للأهداف، ولوحة تتبع ديناميكية في Data Studio تعرض معدلات التحويل وسلوك المستخدم والأداء التقني.  

11. مهمة تحسين عملية النشر الآمن (Integration/Security):
 تنفيذ سير عمل CI/CD آلي يتضمن اختبارات أمان وفحوص تلقائية. مثال ذلك إعداد GitHub Actions أو Jenkins يقوم بفحص الثغرات الأمنية (مثل Snyk، OWASP ZAP) واختبارات الوحدات قبل الدمج. كما يشمل توفير بيئة نشر مبنية حاوية (Docker) لضمان الاستنساخ الآمن للبيئة.  
    المدخلات: ملفات إعداد CI/CD، أدوات الفحص الأمنية، سكريبتات البناء.  
    المخرجات: خط نشر آلي موثق، تقارير فحص السلامة لكل إصدار، ونشر موثوق خالٍ من الثغرات.  

12. مهمة ضمان استقرار الاختبارات وإتمامها (Quality/Integration): 
إصلاح الاختبارات التالفة الحالية وإنشاء اختبارات وحدة واختبارات تكامل جديدة لتغطية الوظائف الحيوية. ثم دمج هذه الاختبارات في نظام CI لمنع الكسر في المستقبل.  
    المدخلات: الأكواد الحالية للاختبارات (Unit/Integration)، إطار الاختبار المستخدم (مثل Jest أو Mocha).  
    المخرجات: مجموعة اختبارات مكتملة، مع تأمين نجاحها ضمن عملية الدمج المستمرة (CI)، وتقارير تغطية الاختبار.  

التقرير التنفيذي

الأداء
	•	قياس وتحسين سرعة التحميل: تنفيذ تدقيق أداء شامل باستخدام أدوات مثل Google PageSpeed Insights (Lighthouse) وWebPageTest لتحديد مشكلات سرعة العرض ومؤشرات Web Vitals (LCP, FID, CLS) ￼ ￼. ثم العمل على تقليل زمن التحميل عن طريق تقنيات مثل تصغير ملفات JS/CSS وضغط الصور واستخدام التخزين المؤقت (Caching).
التأثير: رفع سرعة التحميل يعزز تجربة المستخدم ويؤدي إلى ترتيب أعلى في محركات البحث وزيادة ثقة الزوار.
أدوات مقترحة: Google Lighthouse، WebPageTest، Chrome DevTools.
	•	تقليل حجم الموارد وعدد الطلبات: تجميع (bundle) وضغط (minify) ملفات JavaScript وCSS، واستخدام تنسيقات صور حديثة (WebP/AVIF) واستراتيجيات تحميل كسول (lazy-loading) للوسائط ￼.
التأثير: انخفاض وقت العرض وتحسن Core Web Vitals (LCP, CLS) مما يرفع ترتيب الصفحة ويساهم في بناء ثقة المستخدم.
أدوات مقترحة: Webpack/Babel، أدوات تحليل الشبكة (Network) في المتصفح، Lighthouse.

الفهرسة (SEO)
	•	التدقيق الفني والتحسين: إجراء تدقيق SEO شامل باستخدام Screaming Frog SEO Spider وAhrefs وGoogle Search Console للكشف عن المشاكل التقنية (روابط مكسورة، أخطاء Canonical، hreflang غير مضبوطة) ￼ ￼. ويشمل ذلك إصلاح خرائط الموقع (sitemap) والتأكد من صحة البيانات الوصفية (metadata) وأدلة الصور (og:image).
التأثير: صقل الأساس التقني للموقع يعزز ترتيبه في النتائج ويزيد من ثقة محركات البحث.
أدوات مقترحة: Screaming Frog SEO Spider، Ahrefs، Google Search Console.
	•	تحسين المحتوى الدلالي والسوق المحلي: إثراء المحتوى باللغة العربية بإضافة كلمات مفتاحية وجمل دلالية خاصة بالسوق السعودي داخل العناوين والنصوص، وتحسين وسوم الصور (alt) والبيانات المنظمة (Schema.org) لمكونات الموقع ￼.
التأثير: ظهور أعلى في نتائج البحث المحلية وزيادة نسبة النقرات (CTR) والتفاعل العضوي، مما يساهم في زيادة التحويلات.
أدوات مقترحة: Google Search Console، أدوات تدقيق SEO مثل Ahrefs أو SEMrush.

الأمان
	•	كشف وإزالة المفاتيح السرية: تطبيق فحص ثابت وسينتبار (push protection) على مستودع الكود لاكتشاف أي مفاتيح أو بيانات حساسة مدرجة عن طريق الخطأ ￼ ￼. يمكن استخدام GitLab/GitHub Secret Scanning أو أداة Yelp/detect-secrets لكشف وإخطار الفريق بأي تسريبات قبل النشر.
التأثير: منع أي اختراقات محتملة من خلال حماية المفاتيح المخزنة، مما يعزز ثقة المستخدمين وأمان النظام.
أدوات مقترحة: Yelp/detect-secrets، GitLab/GitHub Secret Detection، التدقيق الدوري بالـ SAST (مثل SonarQube).
	•	فحص الثغرات والأداء الآمن: دمج فحص أمني ديناميكي في خط CI/CD باستخدام أدوات مثل Snyk وOWASP ZAP للتحقق من خلو التطبيق من ثغرات الأمان والمكونات القديمة. كما يوصى بإعداد بروتوكولات أمان (CSP, HSTS) واختبارات اختراق دورية.
التأثير: تعزيز الموثوقية والأمان الكلي للمشروع، وزيادة ثقة العملاء والحماية من المهاجمين.
أدوات مقترحة: Snyk، OWASP ZAP، SonarQube، GitHub Advanced Security.

تجربة المستخدم (UX)
	•	توحيد واجهة المستخدم والتصميم: إنشاء نظام تصميم متكامل (Design System) مع شريط تنقل واحد ونمط موحد للأزرار والصناديق. يشمل ذلك ضمان عمل البحث والشات بنفس التجربة على جميع الصفحات، وتحسين بنية محتوى صفحات الخدمات بإضافة دعوات واضحة للإجراء (CTAs) ونصوص مبسطة.
التأثير: تجربة متسقة تزيد من رضا المستخدم ووضوح القيمة المقدمة، مما يؤدي إلى ارتفاع معدلات التحويل وثقة الزوار.
أدوات مقترحة: Figma/Storybook لتوثيق التصميم، اختبارات A/B وأدوات قياس تجربة المستخدم.
	•	تحسين تدفق التحويل (Conversion Flow): مراجعة وتبسيط مسار المستخدم بحيث تصل جميع الدعوات المهمة (نماذج التواصل، الأزرار الحرجة) بسهولة، وإضفاء تفاعل ديناميكي (مثل نوافذ منبثقة ذكية) عند الحاجة.
التأثير: زيادة احتمالية إكمال الزوار للإجراءات المستهدفة (تحويل)، وتحسين الموثوقية عبر واجهة مستخدم سلسة.
أدوات مقترحة: Hotjar للاختبارات وتسجيل جلسات المستخدم، Google Optimize للتجارب.

التكامل
	•	دمج النظم والخوادم: توحيد خادم التطبيق عن طريق دمج مجلدات backend وserver في نقطة تشغيل واحدة، مما يقلل التكرار وييسر الصيانة. كذلك ينبغي توحيد كود نقاط النهاية (البحث، الشات) وتحديث السكريبتات الملحقة للتشغيل.
التأثير: تقليل تعقيد الصيانة وتسريع تطوير الخصائص الجديدة، وزيادة استقرار المنتج الكلي.
أدوات مقترحة: GitHub Actions أو Jenkins لعمليات التكامل المستمر، Docker للحاويات.
	•	تحديث محرك البحث الداخلي: بناء محرك بحث مشترك لكل الموقع مع فهرس تحديث حي (مثل Elasticsearch أو Algolia) يحل محل الفهرسة الثابتة القديمة. هذا يوفر نتائج أسرع وأكثر دقة في صفحات الخدمة.
التأثير: تحسين تجربة البحث يزيد من تفاعل المستخدمين ويعزز من فرص التحويل (نظرًا لأن الزوار سيجدون المحتوى المطلوب بسهولة أكبر).
أدوات مقترحة: Elasticsearch أو Algolia مع واجهة مستخدم AJAX، Scraping/API لجلب البيانات.

القياس
	•	تحليل الأداء والتحويلات: إعداد Google Analytics 4 لتتبع الأحداث الرئيسية (مثل إرسال نماذج التواصل وبدء الشات)، وربط البيانات بلوحة معلومات في Google Data Studio (Looker Studio) ￼ ￼. ينصح بتحديد الأهداف KPIs الرئيسية (مثل معدل التحويل، معدل الارتداد) وإعداد تقارير دورية لمتابعتها.
التأثير: يوفر رؤية شاملة لسلوك المستخدم ونجاح الحملات، مما يمكّن من تحسين مستمر يحسن معدلات التحويل ويبني ثقة المستثمرين.
أدوات مقترحة: Google Analytics 4، Google Data Studio (Looker Studio)، Google Tag Manager.
	•	رصد الأداء التقني: إنشاء لوحات مؤشرات أداء (Dashboards) لعرض مقاييس تقنية مثل زمن الاستجابة (TTFB)، ونسبة نجاح التحميل على الهواتف المحمولة، ومعدلات خطأ الخادم. يمكن استخدام أدوات مراقبة خارجية (مثلاً Google Webmaster Tools, WebPageTest نتلك التقارير بشكل دوري).
التأثير: رصد مستمر يؤدي إلى معالجة سريعة للمشكلات التقنية، مما يحافظ على سرعة الموقع وثقة المستخدم.
أدوات مقترحة: Google Search Console، WebPageTest.

المراجع والأدوات الموصى بها: Google Lighthouse/PSI ￼، WebPageTest، Screaming Frog SEO Spider ￼، Ahrefs, Google Search Console ￼ ￼، Yelp/detect-secrets ￼، GitLab/GitHub Secret Scanning ￼، Google Analytics 4/Looker Studio ￼، CI/CD (Jenkins/GitHub Actions)، Snyk, OWASP ZAP، إلخ.


أنت مساعد برمجي/سيو متخصص للسوق السعودي. مهمتك: تلقّي قائمة صفحات HTML (بمجلد أو نص HTML لكل صفحة) ثم إنتاج تحسينات جاهزة للتطبيق لرفع معدلات النقر (CTR) في :contentReference[oaicite:5]{index=5} على محركات البحث وواجهات الذكاء الاصطناعي (AI Overviews / Featured Snippets). ردّك يجب أن يكون JSON صالح (UTF-8) بمصفوفة صفحات. لا تكتب شرحًا طويلًا خارج JSON.

لِكل صفحة، قم بالتالي:
1. حلل العنوان الحالي، والوصف، والهيئات schema (JSON-LD)، والـ H1/H2، والمقتطف الأول (first 160–300 كلمة) والعناصر المرئية (img، alt).
2. أنتج **ثلاثة** صيغ مختلفة لعناوين SEO (title) مُغرية للنقر (A/B/C) — كل عنوان:
   - بالعربية الفصحى، موجز، يتضمن الكلمة المفتاحية الأساسية محليًا، وطول لا يتجاوز 70 حرفًا (أو 540 بكسل تقريبًا).
   - واحد على الأقل يحتوي على رقم/نسبة أو حافز ("%"، "خلاصة في 60 ثانية"، "دليل 2026") لرفع CTR.
3. أنتج **ثلاثة** أوصاف ميتا (meta_description) قابلة للنقر (CTA عربي) بطول بين 120–160 حرفًا، تحوي دعوة واضحة للفعل ومطابقة للنية.
4. اكتب **مقتطف (SERP snippet)** قصير جدًا (≤155 حرفًا) قابل للظهور في نتائج الذكاء الاصطناعي ونتائج البحث، يُركز على فائدة المستخدم مباشرةً.
5. أنشئ **FAQ (3 أسئلة وإجابات قصيرة)** مناسبة لعرضها كـ FAQ schema أو مقتطف مقتبس؛ كل إجابة 20–40 كلمة واضحة ومباشرة.
6. اقترح **Open Graph (og:title, og:description)** و**Twitter card** بصيغة جذابة للـ social sharing.
7. حدّث أو اقترح **JSON-LD schema** (Article/Report/Product/FAQ/Breadcrumb) جاهز للحقن، وحافظ على الحقول: headline, datePublished (إذا متوفر), author, publisher, image, mainEntityOfPage.
8. اقترح **3 نصوص بديلة (alt text)** لكل صورة مهمة داخل الصفحة (50–120 حرف).
9. اقترح **3 روابط داخلية (anchor text)** من صفحات أخرى داخل الموقع لزيادة سِلْسلة التمرير (dwell time) وCTR الداخلي.
10. أضف **سطر واحد تبريري (one-line rationale)** لكل اقتراح يشرح لماذا سيفيد CTR (بصيغة عربية قصيرة).
11. صنّف كل اقتراح حسب الأولوية: High / Medium / Low.

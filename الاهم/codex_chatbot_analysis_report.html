  • لا يقرأه محرك Google ضمن المحتوى الفهرسي (لأنه مكون تفاعلي)، ولكنه يقرأ رسالة الترحيب "مرحباً بك في برايت AI" وهذا يعطي إشارة جيدة للدعم المباشر. استخدام aria-hidden="true" وقت الإغلاق سليم لمعايير A11y.
  • الزر مخفي لمدة 3 ثوانٍ CHAT_FAB_DELAY_MS = 3000 مما يقلل وعي المستخدم بوجوده مبكراً.

القسم الثاني: Prompts تنفيذية لـ Codex

✨ Prompt 1 — تنظيف الملفات الميتة (Dead Code Elimination)

احذف ملف `/frontend/js/chat-widget.js` بالكامل وأزل أي إشارة له من أي ملف HTML. هذا الملف يبحث عن IDs غير موجودة (chatToggle, chatWidget, chatMessages, userInput, sendButton) ويخرج فوراً بـ return. لا يؤثر على أي وظيفة. السبب: تقليل HTTP request + حجم التحميل + إزالة الارتباك في الكود.

✨ Prompt 2 — تعطيل ChatWidget الوهمي في Bundle

في ملف `/frontend/js/main.bundle.js` حول السطر 1380، أضف شرطاً قبل إنشاء كلاس ChatWidget لتجنب تنفيذ Constructor بالكامل وتوفير DOM lookups: ```javascript if (!document.querySelector('[data-chat-controller="index-theme"]')) { new ChatWidget(); } ```

✨ Prompt 3 — سد ثغرة XSS (تعقيم رسائل الخطأ)

في ملف `/frontend/js/index-theme.js` أضف دالة تعقيم فوق دالة `mapErrorMessage`: ```javascript function sanitizeText(text) { if (typeof text !== 'string') return ''; return text.replace(/[<>"'&]/g, (char) => { const entities = { '<': '<' , '>' : '>' , '"' : '"' , "'" : ''' , '&' : '&' }; return entities[char] || char; }); } ``` واستبدل قراءة رسالة الخطأ بـ: `const message=sanitizeText(String(error?.message || '' )).toLowerCase();`

✨ Prompt 4 — معالجة التضليل (شارة الإشعار الكاذب)

في ملف `index.html` (سطر 4530)، قم بإخفاء الشارة الحمراء افتراضياً بوضع `style="display:none;"`. ثم في ملف `index-theme.js` أضف الكود التالي لإظهارها ديناميكياً فقط في حال توفر حاجة حقيقية (مثلا بعد 5 ثواني): ```javascript if (chatHistory.length === 0 && chatBadge) { setTimeout(() => { if (!chatWindow.classList.contains('active') && !chatWindow.classList.contains('show')) { chatBadge.style.display = ''; } }, 5000); } ```

✨ Prompt 5 — إصلاح موضع الـ RTL والتوافقية

في ملف `index.html` (سطور زر الشات بوت)، أزل فئات `right-8` و `left-auto` التي تتعارض مع اتجاه الصفحة واستبدلها بستايل متوافق `style="inset-inline-start: 2rem;"`.

✨ Prompt 6 — ضبط حدود الرغوة والذاكرة

المهام المجمعة لأمان الإدخال والموارد: 1- في `index-theme.js` بداخل دالة `sendChatMessage` أوقف الإرسال لو تجاوز طول الرسالة 1000 حرف. 2- أضف `maxlength="1000"` لعنصر حقل الإدخال `` الخاص بالشات في الـ HTML. 3- أخرج `will-change: transform` من محدد `#chatWindow` العام في ملف استايلات Critical CSS وضعه حكراً على حالة الـ `.active` و `.show`.

✨ Prompt 7 — تحسين تجربة الموبايل (Responsive CSS)

في ملفات الـ CSS الرئيسية، أضف استجابة الموبايل لنافذة الشات التي تتجاوز حجم الشاشات الصغيرة: ```css @media (max-width: 420px) { .chat-window { width: calc(100vw - 32px) !important; right: 16px !important; left: 16px !important; bottom: 80px !important; } } ``` وأضف الخصائص `inputmode="text"` و `enterkeyhint="send"` لحقل إدخال الرسائل لدعم الكيبورد في iOS.

✨ Prompt 8 — زرع Schema الـ SEO وربط الداخلي

المهام المجمعة لحصد التفاعل الرقمي: 1- قم بتحديث الـ Meta Description في `index.html` ليحتوي على عبارة: "دعم فوري بالعربي عبر الشات الذكي". 2- أضف رابط ContactPoint لبروتوكول Schema في الصفحة يربط خدمة الشات بأنها وسيلة تواصل "Customer Service". 3- دع الـ API يعيد (Quick Replies) تربط داخلياً لخدمة الأتمتة (`/frontend/pages/smart-automation/`).
ملاحظة الاستراتيجية: إصلاح الجوانب الثقيلة في كود الشات العائم (Dead Code والـ Excessive CPU load) سيساهم بتمرير متطلبات أداء جوجل للحصول على تصنيف أسرع بكثير. والأهم سد ثغرة XSS التي قد تشكل خطراً بالغاً على الزوار.

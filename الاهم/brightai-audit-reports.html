إصلاح CORS وأمان Backend

Security
في ملف backend/server.js، قم بتعديل إعداد CORS بحيث لا يسمح إلا لنطاقات محددة. 1. ابحث عن الثابت CORS_HEADERS الذي يحتوي 'Access-Control-Allow-Origin': '*' 2. استبدله بمنطق ديناميكي يسمح فقط لـ: - https://brightai.site - https://www.brightai.site - http://localhost:3000 (في بيئة التطوير فقط) 3. أضف Origin checking: إذا كان الطلب من origin غير مسموح، أرجع 403 مع رسالة خطأ واضحة 4. أضف rate limiting headers في جميع الاستجابات 5. في نفس الملف، أزل أي hardcoded URLs مثل 'extraordinary-axolotl-d35933.netlify.app' واستبدلها بـ process.env.BRIGHTAI_API_BASE 6. أضف validation للـ Origin header قبل معالجة أي طلب API تأكد من اختبار التغييرات مع: - طلب من brightai.site (يجب أن ينجح) - طلب من domain غير مصرح (يجب أن يُرفض بـ 403)
🔍

إصلاح GSC وBing Webmaster Verification

SEO
في ملف index.html، افحص جميع meta tags التي تحتوي على قيم placeholder وأصلحها: 1. ابحث عن: content="REPLACE_WITH_YOUR_GSC_VERIFICATION_CODE" - استبدل بـ: content="[GSC_CODE]" — أدخل هنا رمز التحقق من Google Search Console الخاص بـ brightai.site 2. ابحث عن: content="REPLACE_WITH_YOUR_BING_CODE" - استبدل بـ: content="[BING_CODE]" — أدخل هنا رمز Bing Webmaster Tools 3. أنشئ سكريبت Node.js في scripts/validate-seo-placeholders.mjs يفحص جميع ملفات HTML ويبحث عن: - أي content تحتوي REPLACE_ أو YOUR_ أو PLACEHOLDER - أي href تحتوي example أو localhost في production - يطبع تقريراً بالملفات المتضررة 4. أضف هذا الفحص لـ CI workflow في .github/workflows/ci-tests.yml كـ job منفصل "seo-placeholder-check" 5. في ملف robots.txt، حدّث قاعدة Allow: استبدل: Allow: /frontend/ بـ: Allow: /frontend/css/ Allow: /frontend/js/ وأزل Allow: /frontend/ العامة

🚨 Sentry

تتبع الأخطاء في Frontend وBackend لحظياً — اعرف عن crash الموقع قبل أن يشكو العميل

لماذا الآن؟

لديك @sentry/react في package.json غير مفعّل
Backend بلا error tracking
5K errors/شهر مجاناً
Alert فوري على الإيميل

⚡ Upstash Redis

استبدل الـ in-memory session store بـ Redis حقيقي — ضرورة عند أول scale للـ backend

المشكلة الحالية؟

Sessions تختفي عند restart
Rate limiter يُفقد بياناته
لا يعمل مع multiple instances
Cache الـ AI يُفقد كذلك

🗺️ ما الذي يتغير؟

الحالة الحالية: in-memory Map
← تختفي عند restart
Backend Node.js
→ REST API →
Upstash Redis
→
Sessions ✓ Cache ✓ RateLimit ✓

// Session Store — Upstash Redis مع fallback للذاكرة
const URL   = process.env.UPSTASH_REDIS_REST_URL;
const TOKEN = process.env.UPSTASH_REDIS_REST_TOKEN;
const TTL   = (parseInt(process.env.CHAT_SESSION_TTL_MS,10)||600000)/1000;
const mem   = new Map();
const live  = !!(URL && TOKEN);

async function r(cmd,...args) {
  const res = await fetch(
    `${URL}/${cmd}/${args.map(encodeURIComponent).join('/')}`,
    {headers:{Authorization:`Bearer ${TOKEN}`}}
  );
  return (await res.json()).result;
}

module.exports = {
  async get(k) {
    if (!live) return mem.get(k)??null;
    const v = await r("GET",k);
    return v ? JSON.parse(v) : null;
  },
  async set(k,v) {
    if (!live){mem.set(k,v);return;}
    await r("SET",k,JSON.stringify(v),"EX",TTL);
  },
  async del(k) {
    if (!live){mem.delete(k);return;}
    await r("DEL",k);
  },
  isRedis: ()=>live,
};

⚡ Codex ينفذها
Cache لاستجابات Gemini AI
📄 backend/utils/aiCache.js (ملف جديد)
Node.js
// يوفر تكلفة Gemini API للأسئلة المتكررة
const store  = require("./sessionStore");
const crypto = require("crypto");

function key(p) {
  return "ai:"+crypto.createHash("sha256")
    .update(p.trim().toLowerCase())
    .digest("hex").slice(0,16);
}

module.exports = {
  async get(p)   { return await store.get(key(p)); },
  async set(p,r) {
    if ((r?.length||0) > 50)
      await store.set(key(p), {response:r, cached:true, t:Date.now()});
  },
};

عدّل backend لمشروع BrightAI لاستخدام Upstash Redis: 1. أعد كتابة backend/utils/sessionStore.js بالكامل: - إذا توفرت UPSTASH_REDIS_REST_URL وUPSTASH_REDIS_REST_TOKEN: استخدم Redis REST API بـ fetch() - وإلا: استخدم Map في الذاكرة كـ fallback (لا تكسر الكود الحالي) - وفّر: async get(key)، async set(key, value)، async del(key)، isRedis() - TTL من CHAT_SESSION_TTL_MS / 1000 أو 600 ثانية كـ default 2. عدّل backend/middleware/rateLimiter.js: - استخدم store.get/set بدلاً من Map مباشرة - Key: "rl:{IP}:{Math.floor(Date.now()/60000)}" - أضف headers: X-RateLimit-Limit وX-RateLimit-Remaining 3. أنشئ backend/utils/aiCache.js: - hashKey(text): "ai:" + أول 16 حرف SHA256 - get(prompt): store.get(hashKey(prompt)) - set(prompt, response): يخزن فقط إذا response.length > 50 مع TTL 86400 4. في backend/routes/gemini.js، أضف قبل استدعاء Gemini API: const cached = await aiCache.get(userMessage); if (cached) return res.json({...cached, fromCache: true}); ثم بعد الحصول على الرد: await aiCache.set(userMessage, responseText); 5. في backend/server.js، أضف endpoint /health يتحقق من Redis: const redisOk = store.isRedis() ? await store.get("__health__").then(()=>true).catch(()=>false) : "memory-mode"; res.json({ status: "ok", redis: redisOk, uptime: process.uptime() }); 6. في backend/.env.example أضف: UPSTASH_REDIS_REST_URL=https://your-db.upstash.io UPSTASH_REDIS_REST_TOKEN=your_token 7. في .github/workflows/ci-tests.yml أضف للـ backend-tests job: env: UPSTASH_REDIS_REST_URL: "" UPSTASH_REDIS_REST_TOKEN: "" (قيم فارغة تجعل الكود يستخدم الـ fallback في CI) ملاحظة: UPSTASH_REDIS_REST_URL وTOKEN يُنشآن من console.upstash.com ثم يُضافان لـ Netlify Environment Variables

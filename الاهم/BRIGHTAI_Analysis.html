١. نظرة عامة على المشروع
BrightAI منصة سعودية متخصصة في تقديم حلول الذكاء الاصطناعي للشركات والقطاعات التجارية. تستهدف الشركات السعودية وتدعم رؤية 2030، مع تقديم ذكاء اصطناعي عبر وكلاء (AI Agents) قابلين للتخصيص، ومدونة محتوى ضخمة، وأدوات تحليل بيانات وأرشفة طبية ذكية.

الموقع المنشور: https://brightai.site | يعمل على Netlify مع رفع تلقائي عبر GitHub Actions.


📁 frontend
موقع ويب ستاتيكي (HTML/CSS/JS) — صفحات تسويقية، مدونة، توثيق، أدوات تجريبية
⚙️ backend
Node.js API Gateway — تكامل مع Groq, Gemini, RAG بحث، streaming، OCR، ذاكرة طبية
🚀 platform
React + TypeScript SPA — لوحة تحكم، منشئ وكلاء الذكاء الاصطناعي، سوق الوكلاء، تحليلات

٢. الهيكل المعماري

الواجهة الأمامية (Frontend)

  • نوعان من الواجهة: موقع ستاتيكي قديم (HTML/CSS/JS) + منصة React حديثة
  • CSS Framework: TailwindCSS مع دعم RTL
  • الحركات: GSAP, AOS, Framer Motion
  • الأيقونات: Iconify مع preload مُحسَّن
  • Service Worker: sw.js لدعم العمل أوفلاين
  • PWA: manifest.json مُهيَّأ
  • Routing: React Router v6

الخلفية (Backend)

  • Runtime: Node.js بدون Framework (HTTP خام)
  • WebSocket: ws للبث الحي في الوقت الفعلي
  • AI Provider: Groq (Llama, Gemma) + Google Gemini
  • RAG Search: بحث دلالي مبني على JSON index
  • Rate Limiting: Middleware مخصص بالذاكرة
  • Session Store: In-memory مع تنظيف دوري
  • Caching: نظام cache مخصص

منصة React (brightai-platform)

  • State Management: Zustand للـ workflow + React hooks
  • Database: Supabase (PostgreSQL مع RLS)
  • Auth: Supabase Auth + OAuth + Local Admin Mode
  • Workflow Engine: ReactFlow لرسم الوكلاء
  • Internationalization: i18next (عربي + إنجليزي)
  • Charts: Recharts
  • Encryption: Web Crypto API للمفاتيح

النشر والـ DevOps

  • Hosting: Netlify (source of truth)
  • CI/CD: GitHub Actions (ci-tests + netlify check)
  • Domain: brightai.site مع redirect من brightaii.com
  • Docker: Dockerfile + docker-compose للنشر المحلي
  • Nginx: nginx.conf مُهيَّأ للإنتاج
  • Serverless: Netlify Functions لـ AI, Audit

🏗️ منشئ الوكلاء (Agent Builder)

واجهة 5 خطوات لبناء وكلاء الذكاء الاصطناعي:

  • الخطوة 1: المعلومات الأساسية (اسم، وصف، فئة، أيقونة)
  • الخطوة 2: الشخصية (لهجة، قواعد سلوك، قاعدة معرفة)
  • الخطوة 3: مسار العمل (Workflow بالـ ReactFlow)
  • الخطوة 4: الإعدادات المتقدمة (نموذج AI، webhook، خصوصية)
  • الخطوة 5: الاختبار والنشر

يدعم المسودات التلقائية، استعادة المسودة، واختصارات لوحة المفاتيح

📊 لوحة التحكم والميزات

  • Dashboard: إحصاءات، رسوم بيانية، نشاط حي، تنبيهات
  • Marketplace: سوق الوكلاء مع تقييمات ونشر
  • Analytics: تتبع أداء تفصيلي للوكلاء
  • Audit Logs: سجل أمني كامل للعمليات
  • Settings: ملف شخصي، مفاتيح API، أمان، إشعارات، باقات
  • Templates: قوالب جاهزة لتسريع الإنشاء
  • Node Editor: محرر العقد المتقدم (Groq, condition, loop, action)

🧠 نظام تنفيذ الوكلاء

AI Orchestrator

تخطيط خطوات تلقائي → تحسين الـ prompt → تنفيذ → تقييم جودة (min score 0.72) → إعادة محاولة

Workflow Runner

تنفيذ عقد متسلسل/متوازي مع timeout، تتبع نتائج كل عقدة، وكلاء متداخلة (depth tracking)

Agent Memory

ذاكرة دلالية محفوظة في Supabase، تُسترجع بناءً على تشابه الرسائل لتوفير سياق المحادثات السابقة

⚠️ نقاط تحتاج تحسين

⚠️
ملفات مكررة: وجود search 2.js, search 3.js, cache 2.js — يدل على بقايا تحرير يدوي غير مُنظَّف
⚠️
Backend بدون Framework: HTTP handler يدوي بدلاً من Express/Fastify — يصعّب الصيانة مع النمو
⚠️
Session Store في الذاكرة: لا يتحمل multi-instance أو إعادة تشغيل الـ server
⚠️
Rate Limiter في الذاكرة: نفس مشكلة عدم التوافق مع التوسع الأفقي
⚠️
أسماء TypeScript بالعربية: مثير للجدل — جيد للمشروع المحلي لكن يُصعّب التوظيف الدولي
⚠️
Backend dependencies محدودة: package.json يذكر فقط dotenv وws — بقية الحزم مثل Groq SDK غير مذكورة صراحة
⚠️
Platform SPA بدون SSR: يؤثر على SEO للصفحات الديناميكية

🛡️ طبقات الأمان المُطبَّقة

  • RLS في Supabase: حماية على مستوى الصف لجميع الجداول الـ 17
  • تشفير مفاتيح API: Web Crypto API قبل التخزين — لا تشفير مزدوج
  • Security Headers: ملف _headers مع CORS, CSP, X-Frame-Options
  • Input Sanitization: sanitizer.js لجميع مدخلات المستخدمين
  • Rate Limiting: Middleware لحماية API endpoints
  • Audit Logs: سجل أمني كامل مخزون في Supabase
  • HTTPS فقط: Netlify يفرض HTTPS + redirect من HTTP
  • Security Audit Migration: 00002_security_audit.sql
